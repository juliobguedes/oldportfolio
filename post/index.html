Tormenta Spellbook

Tormenta is a brazilian RPG based on D&D (Open Game License was a bless) very similar to D&D3.5 but with a totally different and new scene. The spells and feats are very similar, having a few differences in the mechanics and rules.

As a RPG player and master, a great part of the time playing was devoted to search stuff in the books during the game. In TormentaRPG, we have a lot of them, and had even more in D&D. To make the RPG sessions faster and more fluid, I adapted the RoenMidnight’s Grimório TRPG to Angular.js, being even easier to adapt and add features.

So, RoenMidnight came up with the idea, based on an existing grimoir for D&D, to make an online grimoir for Tormenta. I translated the entire work that Roen made and added a few of the feats present in Tormenta in a section that didn’t exist in RoenMidnight’s grimoir. Also, to make my version unique, I changed the name of the website to Tormenta Spellbook, that can be accessed in the image below.

First of all, it is important to say that i made this project only for fun and to practice what I already knew about Angular.js. Second, I made this in the most basic way to do it, and the most sophisticated thing i did was nesting views and promise chaining. Also, I changed the original data using simple python scripts (didn’t even knew how to properly use python libs back then) managing json in pure string. I’ll soon be updating the scripts and adding proper comments to them.

Hope all the RPG players keep the dices rolling, and lets play!

MusicHub

Easy Stock is a web application developed in Angular.js. The application was made as a part of the Informational Systems class, taught by Melina Mongiovi.

Main user stories:

  1. As an user, I would like to add an artist to the system. In a text box, the user can add the artist info and add him to the system.
    • The artist name is required, but its image is optional;
    • Two different artists can not have the same name;
  2. As an user, I would like to add a song to the system. In a text box, the user can add the music info and add it to the system.
    • The music info are: name, artist, album, release year and length. All info is mandatory;
    • If an album with the same name exists, the song must be added in it;
    • An album can not have two songs with the same name;
  3. As an user, I would like to search artist in the system. In the home page, the user must be able to search for an artist.
    • Clicking in an artist, must be possible to see its name, album list, image and last listened song by that artist. More info can be included;
    • A field to the artist rating is required;
    • Is required a method to register the last listened song of an artist;
  4. As an user, I would like to mark existing artist as favorites. The favorite artists must be shown in the home page. There must be also a method to unmark the artist as favorite;
  5. As an user, I would like to create playlists and add or remove existing songs to it;
    • Clicking in a playlist should show its songs;
    • Deleting a playlist is also required;
    • Two playlists can not have the same name;

Based on those user stories, MusicHub was created. You can check out the website clicking in the image below.

Easy Stock

Easy Stock is a web application developed in Angular.js, Spring-boot and PostgreSQL. The application was made as a part of the Informational Systems class, taught by Melina Mongiovi.

During the class, we learned code patterns and, in the end of the class, we had to develop (and fix) a project given by Melina. The project had 13 main user stories and 3 aditional user stories:

Main user stories:

  1. As an admin, I would like to have the system persisting all the data;
  2. As an admin, I would like to login in the system to have access to all the admin funcionalities;
  3. As an admin, I would like to access the system through a link;
  4. I, as a client, would like to consult the disponibility and price of each product of the supermarket without the need of being logged in;
  5. As an admin, I would like to order the products and the record of sales according to their important information (i.e. name, price, etc);
  6. As an admin, I would like to set up discounts to the product categories, in a way that the costumer receives it in the end of the buying process. The discount categories are: 0% (no discount), 10% (good), 25% (great) and 50% (super);
  7. As an admin, I would like to see a general report of the supermarket, with the products and their data (quantity, number of packs, expiration date, etc), sales record and profit;
  8. As an admin, I would like to see the product marked as unavailable if it has zero items or if its expired. Also, if its unavailable, the system doesn’t show its price and the product is added, if its expired, should be added to the list of expired products;
  9. As an admin, I would like to register a sale of a product, recording the value of the purchase. The product quantity must be decremented;
  10. As an admin, I would like to cancel a sale of a product, deleting the record of the sale. The product quantity must be incremented;
  11. As an admin, I would like to receive notifications in the home screen warning if a product is under 15 units;
  12. As an admin, I would like to receive notifications in the home screen warning if a product is about to expire (1 month or less);

Aditional user stories:

  1. As a client, I would like to sign up to access the system;
  2. As a client, I would like to log in to access the client funcionalities;
  3. As a client, I would like to bookmark my shopping list in the system, so my purchases would be collected and packaged in advance.

We made a group then, with me, Iann, Livia and Marcus. Initially, the system didn’t follow almost any code patterns, and we had to fix and put the correct patterns in the code (both front and back). After that, we started implementing the new features (given in the user stories above) and our group subdivided in 2 lesser groups: Livia and Marcus in the backend, me and Iann in the frontend.

We started by refactoring the whole project to have less imports in each controller and separating the angular controllers. After that, the implementation of services to hold the http requests, passed to the controllers in promise chaining. After that, we reimplemented the whole system to work only in one language (previously portuguese and english mixed up): english.

The result code can be found in Livia’s repository with our contributions.

Libratizar

Libratizar is a Startup from Campina Grande with focus in inclusive education. Our main proposal is that education should be to everyone.

The Libratizar started in Campina Grande in the 14th FETECh, organized by the Parque Tecnológico/PB (PaqTecPB) and the Incubadora Tecnológica de Campina Grande (ITCG), during the 1st FETECh Hackathon. I made my subscription to the event with 2 other friends, Renan and Mariana, but the teams were drawn in the beginning of the competition, revealed with the theme.

After a while, the team was revealed: Me, Kalina Ferreira and Samuel Rufino. The theme of the competition was impact business in education. We watched a few lectures by Marcel Fukayama from Din4mo, Hyggo Almeida, Danilo Santos and Veronica Macario.

We spent a lot of time thinking what our team would do based on the theme of the competition. A few not so great - still valid - ideas were coming up, and I had a very primitive idea. The rest of the team kept the discussion about the existing ideas going and I started walking and writing in a piece of paper, while listening to music, trying to refine it.

After 15 or 20 minutes, I came back and the idea was ready: We should do a startup focused in the education of deaf children, something like an online platform where kids all around the country could access video lectures made by pedagogical teachers and taught in LIBRAS (Brasilian Language of Signs). We included a gamification learning process, studied statistics, came up with investment needing to put the startup in the market and a lot more. Kalina made the mocks of the pages, Samuel made the use case diagram and I started programming an example of what the page would be using Angular.js.

48 hours later, we had a name, a logo and a purpose: to make Libratizar a real startup in education: every brazilian deaf children should be able to write and read portuguese. In the final pitch I was chosen to present our startup and that was it. The hackathon was finished and the final standings were revealed, and the competition that should only have one winner had changed: unfortunately, we didn’t win, but still, we were in second place and it was amazing for us.

A month later, we received the invitation to present our startup in the 3rd Startup Workshop of Campina Grande and again I presented our startup. It was a great honor for all of us to do such a thing.

Data Visualizations

Hey everyone!

My data visualization page was created using Hugo to host a series of visualizations made in d3.js and Vega Lite, a d3 compiler. All of them were made in the DataVis class @ UFCG, taught by Nazareno Andrade, with the purpose to learn d3.js, the tecniques and theory about visualizations.

In each post, I’ll present the same guide made by Nazareno that lead me to create the visualizations. I a few of them I made use of Python to modify the original data. The Python scripts are simple and shouldn’t be hard for you to understand them. Both data and the scripts are (or soon will be) linked to the post.

You all should know that d3.js is a powerful tool to make visualizations, so feel free to change the data and the visualization to learn more from it.

Hope you all enjoy it!
