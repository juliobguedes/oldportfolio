Easy Stock

Easy Stock is a web application developed in Angular.js, Spring-boot and PostgreSQL. The application was made as a part of the Informational Systems class, taught by Melina Mongiovi.

During the class, we learned code patterns and, in the end of the class, we had to develop (and fix) a project given by Melina. The project had 13 main user stories and 3 aditional user stories:

Main user stories:

  1. As an admin, I would like to have the system persisting all the data;
  2. As an admin, I would like to login in the system to have access to all the admin funcionalities;
  3. As an admin, I would like to access the system through a link;
  4. I, as a client, would like to consult the disponibility and price of each product of the supermarket without the need of being logged in;
  5. As an admin, I would like to order the products and the record of sales according to their important information (i.e. name, price, etc);
  6. As an admin, I would like to set up discounts to the product categories, in a way that the costumer receives it in the end of the buying process. The discount categories are: 0% (no discount), 10% (good), 25% (great) and 50% (super);
  7. As an admin, I would like to see a general report of the supermarket, with the products and their data (quantity, number of packs, expiration date, etc), sales record and profit;
  8. As an admin, I would like to see the product marked as unavailable if it has zero items or if its expired. Also, if its unavailable, the system doesn’t show its price and the product is added, if its expired, should be added to the list of expired products;
  9. As an admin, I would like to register a sale of a product, recording the value of the purchase. The product quantity must be decremented;
  10. As an admin, I would like to cancel a sale of a product, deleting the record of the sale. The product quantity must be incremented;
  11. As an admin, I would like to receive notifications in the home screen warning if a product is under 15 units;
  12. As an admin, I would like to receive notifications in the home screen warning if a product is about to expire (1 month or less);

Aditional user stories:

  1. As a client, I would like to sign up to access the system;
  2. As a client, I would like to log in to access the client funcionalities;
  3. As a client, I would like to bookmark my shopping list in the system, so my purchases would be collected and packaged in advance.

We made a group then, with me, Iann, Livia and Marcus. Initially, the system didn’t follow almost any code patterns, and we had to fix and put the correct patterns in the code (both front and back). After that, we started implementing the new features (given in the user stories above) and our group subdivided in 2 lesser groups: Livia and Marcus in the backend, me and Iann in the frontend.

We started by refactoring the whole project to have less imports in each controller and separating the angular controllers. After that, the implementation of services to hold the http requests, passed to the controllers in promise chaining. After that, we reimplemented the whole system to work only in one language (previously portuguese and english mixed up): english.

The result code can be found in Livia’s repository with our contributions.
